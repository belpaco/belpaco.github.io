Introduction

Web Application Development is an in-depth exploration into the complicated world of structuring content in a logical and progressive way. It's an important course that covers core concepts and abilities required for creating dynamic and user-friendly web apps. As aspiring information technology developers, it is critical to understand the fundamentals of this field, since it serves as the foundation for current web development.

  • HTML, or HyperText Markup Language, serves as the backbone of the web, providing the structure for web pages. It utilizes tags to define elements such as headings, paragraphs, images, and links, enabling browsers to interpret and display content accurately. HTML lays the foundation for web development, offering a standardized way to organize and present information on the internet.
  • CSS, short for Cascading Style Sheets, complements HTML by controlling the visual presentation of web pages. It allows developers to define styles such as colors, fonts, layouts, and animations, enhancing the aesthetics and usability of websites. CSS operates on a cascading principle, where styles can be inherited, overridden, or cascaded down from parent to child elements, providing flexibility and consistency in design.
  • JavaScript (JS) is a dynamic programming language that adds interactivity and functionality to web pages. It enables developers to create responsive and interactive elements such as forms, animations, and dynamic content updates. JavaScript can manipulate the HTML and CSS of a webpage in real-time, responding to user actions and events, making it an indispensable tool for creating engaging and interactive web experiences.
What You Should Already Know

Here's a short refresher of the basics in HTML5, CSS, and JavaScript:

  • HTML is the standard markup language used to create web pages. It consists of a series of elements, enclosed in tags, which structure the content of a webpage. The basic structure of an HTML document includes the <!DOCTYPE html> declaration, followed by the <html>, <head>, and <body> tags. Within the body, various elements such as headings (<h1> to <h6>), paragraphs (<p>), images (<img>), links (<a>), and lists (<ul>, <ol>, <li>) are used to organize and present content. Attributes can be added to elements to provide additional information or functionality, such as the href attribute for links or the src attribute for images.
  • CSS is a style sheet language used to control the presentation and layout of HTML documents. It allows developers to define styles for elements such as colors, fonts, margins, padding, and positioning. CSS rules consist of a selector that targets HTML elements and a declaration block that specifies the styles to apply. Selectors can target elements based on their type, class, id, or relationship to other elements. Styles can be defined inline within HTML elements, in a <style> element within the document's <head>, or in an external CSS file linked to the HTML document using the <link> element.
  • JavaScript is a high-level, dynamic programming language primarily used for adding interactivity and behavior to web pages. It allows developers to manipulate the HTML and CSS of a webpage in real-time, respond to user actions and events, and dynamically update content without reloading the entire page. JavaScript code is typically embedded within HTML documents using <script> tags or included from external JavaScript files. It supports a wide range of features, including variables, data types, operators, control structures (such as loops and conditionals), functions, objects, arrays, and more. JavaScript is often used in conjunction with HTML and CSS to create rich, interactive web experiences.
Client-side vs. Server-side Development

Client-Side Development:

  • Definition: Activities on the user's device (like a web browser) focusing on the interface and user experience.
  • Technologies: HTML, CSS, JavaScript, plus frameworks like React or Angular.
  • Responsibilities:
    • Structuring web pages (HTML)
    • Styling them (CSS)
    • Adding interactivity (JavaScript)
  • Advantages:
    • Quick and responsive interfaces
    • Less strain on servers
    • Dynamic user experience

Server-Side Development:

  • Definition: Activities on the web server managing data, requests, and server operations.
  • Technologies: Languages like PHP, Python (Django), or JavaScript (Node.js).
  • Responsibilities:
    • Data handling
    • Business logic
    • User authentication
    • Generating content
  • Advantages:
    • Better security
    • Centralized control
    • Scalability

Interaction Between Client-Side and Server-Side:

  • Communication: They talk through HTTP(S) using formats like JSON or XML.
  • Workflow: Clients request, servers respond with data or actions.

Full-Stack Development:

  • Definition: Developers handling both client and server aspects.
  • Responsibilities: Versatile skills, from interface design to server management.
Git
Git is a distributed version control system (DVCS) designed to track changes in source code during software development. It was created by Linus Torvalds in 2005 to manage the development of the Linux kernel but has since become widely adopted across various industries and projects.

Installation

  1. Go to the official Git website here.
  2. Download the appropriate version for your operating system (Windows, macOS, Linux).
  3. Run the installer and follow the on-screen instructions.
  4. During installation, you can choose the default settings or customize them according to your preferences.
  5. Once the installation is complete, you can verify it by opening a terminal or command prompt and typing git --version. If Git is installed correctly, it will display the installed version.

Cloning a Repository

  1. Open a terminal or command prompt.
  2. Navigate to the directory where you want to clone the repository using the cd command.
    3. ex. cd cc14 or cd <directory>
  3. Copy the URL of the repository you want to clone from a platform like GitHub.
  4. In the terminal, type git clone followed by the repository URL.
    5. ex. git clone https://github.com/<username>/<repository>
    or git clone https://github.com/belpaco/225353826-WebApp
  5. Press Enter. Git will download the repository files to your local machine.

Basic Git Commands

  • git init: Initializes a new Git repository in the current directory.
  • git add <file>: Adds a file to the staging area, ready to be committed.
  • git commit -m "commit message": Commits the staged changes with a descriptive message.
  • git status: Shows the status of files in the working directory and staging area.
  • git push: Uploads local changes to the remote repository.
  • git pull: Downloads changes from the remote repository to your local machine.
  • git branch: Lists all branches in the repository.
  • git checkout <branch>: Switches to the specified branch.
  • git merge <branch>: Merges changes from the specified branch into the current branch.
  • git log: Displays the commit history of the repository.

More on Cascading Style Sheets

CSS Style Rules

Selectors

Definition: Selectors target HTML elements to which the styles will be applied.

Examples: body targets the <body> element. .classname targets elements with a specific class name #id targets elements with a specific ID.

Properties and Values

Definition: Properties define the aspect of the element to be styled, while values specify the style's characteristics.

Examples: color: red; sets the text color to red. font-size: 16px; sets the font size to 16 pixels background-color: #FFFFFF; sets the background color to white.

Declaration Block

Definition: A group of one or more property-value pairs enclosed within curly braces {}.

            h1 {
              color: blue;
              font-size: 24px;}

This declaration block styles <h1> elements with blue text and a font size of 24 pixels.
